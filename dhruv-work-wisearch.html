WiSearch

An Intelligent Semantic Search Engine designed to solve the "vocabulary mismatch" problem in academic research, enabling the retrieval of contextually relevant literature based on user intent and meaning rather than exact keyword matches.

My Role & Contributions

  • Vector Embeddings Pipeline – Utilized Sentence Transformers (BERT-based models) to compute dense vector embeddings of academic papers.
  • Efficient Retrieval – Integrated FAISS (Facebook AI Similarity Search) index for high-speed, sub-millisecond nearest-neighbor search.
  • Interactive Interface – Developed a responsive React and TypeScript frontend to view matches, browse similarity scores, and visualize semantic space.
  • API Layer – Built and documented FastAPI endpoints to process raw queries and return JSON-formatted literature candidates.

Tech Stack

Python, FastAPI, FAISS, Sentence Transformers (BERT), React, TypeScript.

Outcome

Created a working semantic search system that mitigates keyword-only limitations, demonstrating how modern vector embeddings can significantly improve information retrieval in scientific databases.
